# Backlog Maintenance — Complete Removal of Resolved Entries

## Backlog Maintenance – Completion Removal

BACKLOG.md is a work queue for things that need to be worked on. Completed backlog entries must be removed entirely rather than marked as DONE or annotated with verbose notes. The goal is to keep the backlog current and, once all work is complete, to have an empty backlog. An appendix in BACKLOG.md lists verified-fixed items so Opus reviews do not re-flag them. Deprecation banners were added to docs/arch-review-queue.md redirecting to BACKLOG.md Sections 1 and 4. Deprecation banners were added to WIP.md redirecting to BACKLOG.md Section 2, marking prior entries as merged or abandoned.
