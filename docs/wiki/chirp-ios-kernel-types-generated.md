# Chirp iOS KernelTypes.generated.swift — Dev-Time Generation, Lives in Git

> KernelTypes.generated.swift is generated by a dev-time tool (not at build time), lives in git, and has precedent for manual field additions — it is not a build artifact like protobuf/flatbuffers generated code.

## Generation Model

`KernelTypes.generated.swift` is generated by a dev-time tool, not at build time. Unlike protobuf or FlatBuffers generated code that is regenerated on every build, this file is generated once by a developer tool and committed to the repository. It lives in git and is treated as a source file, not a build artifact. [^38935-8]

## Manual Editing Precedent

The codebase has established precedent for manually adding fields to `KernelTypes.generated.swift`. When the kernel emits new projection fields (e.g., `claimed_events`), the corresponding Swift types (`ClaimedEventDto` + `claimedEvents: [String: ClaimedEventDto]?` on `SnapshotProjections`) can be added directly to the generated file. This is a Swift-only edit — no Rust codegen change is needed for the consuming Swift side to receive new projection data that the kernel already emits. [^38935-9]

## Relationship to Rust Codegen

The Rust side emits projection data that the C FFI already surfaces (e.g., `nmp_app_claim_event` / `nmp_app_release_event` are in `NmpCore.h`). When the kernel supports a projection and the C FFI symbols exist, the Swift side can consume it by adding the corresponding DTO types to `KernelTypes.generated.swift`. No Rust change is required — it's a pure Swift decoding exercise. [^38935-10]

## Distinct From Build-Time Codegen

This is distinct from FlatBuffers schema codegen (`flatc`-generated bindings), which is regenerated on every schema change and must stay in sync with the runtime library version. `KernelTypes.generated.swift` has no such build-time dependency — it's a snapshot of the kernel's projection shape at dev time, kept in git alongside other Swift source files. [^38935-11]


Manual Editing Precedent

In PR #795, a rebase agent discovered that `claimed_events` was a live Rust projection present in the kernel but missing from `KernelTypes.generated.swift`'s `SnapshotProjections`. The agent registered it (moving from 32 to 33 entries), moved `ClaimedEventDto` to `EmbedHost.swift`, and regenerated — this is the established pattern for adding missing projection types to the generated file when the kernel already emits them.
