## Limitations

- This is a deterministic prototype harness. Real relay sockets, LMDB/SQLite writes, UniFFI marshaling, platform wrappers, and NSE calls are modeled because those runtime pieces do not exist yet.
- Replay mode is CI-shaped and deterministic. Capture mode currently writes a synthetic trace manifest, not live WebSocket frames.
- Live mode is intentionally not implemented until the actor, relay adapter, and storage backend exist.

## Observations

- Prototype replay gates pass, but they mainly validate the planned budgets and wrapper/cache lifecycle model, not real I/O behavior.
- The highest-risk unmeasured surfaces are durable storage latency, real relay burstiness, UniFFI/platform-shadow memory, and actual NIP-77 support.
- Next implementation step: replace synthetic storage/relay models with the real actor, relay adapter, storage backend, and generated wrapper shadow.
