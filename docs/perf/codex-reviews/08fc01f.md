## REPORT (orchestrator follow-up)

- **D7/public-API: AckClass on dispatcher boundary.** `RelayAck::Failed { class: AckClass, .. }` puts classification on the dispatcher's wire, which contradicts the trait doc "the dispatcher returns raw transport results; classification + retry policy live in the engine." Suggested follow-up: make `RelayAck::Failed` carry raw `{ message, code }` (or just the NOTICE/CLOSED frame text); add an internal `classify_ack(message: &str) → AckClass` in the engine; backfill tests to lock the classification table. Touches public API, requires new tests → not codex-cycle scope.

- **Durable retry queue incomplete.** `PublishStore` persists only `per_relay`, not `pending_retries` (the retry-deadline-by-relay map). After "restart" (`resume_from_store`), retries fire immediately without honouring the original backoff. For the in-memory test this is acceptable; the LMDB-backed impl in M3 needs to persist + restore `pending_retries`. Suggested follow-up when M3 LMDB row schema lands: extend `PublishRecord` with `Vec<(RelayUrl, u64)>` for `pending_retries`; add round-trip test.

- **AUTH-REQUIRED reauth path not wired to Signer.** `apply_verdict::Reauth` models reauth as immediate retry; `Signer` is currently unused (`#[allow(dead_code)]`). Will be closed when M6 ledger lands and the bridge calls `signer.sign_auth(challenge, relay_url)` between verdict and dispatch.

- **File-size soft cap.** `crates/nmp-core/src/publish/tests.rs` 338 LOC, `crates/nmp-core/tests/publish_engine.rs` 390 LOC. Both under 500 hard cap; precedent `crates/nmp-testing/tests/m2_subscription_compilation_audit.rs` is 460 LOC. Not split — cohesive scenario coverage. Codex flagged for AGENTS 300 LOC discipline; orchestrator decided not to split to keep test scenarios grouped by concern.

- **`publish_durable_across_restart` is in-memory.** Two engine instances share one `Arc<InMemoryPublishStore>` — that's two engines reading the same `Mutex<HashMap>`, not a serialize/deserialize round-trip through actual storage. The M3 LMDB-backed `PublishStore` impl needs its own round-trip test to fully close this gap. Logged in PD-003.
