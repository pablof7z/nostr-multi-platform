1. **Kernel dispatch hook absent (known Phase-1 deferral).**
   `crates/nmp-core/src/substrate/domain.rs:21`,
   `crates/nmp-core/src/substrate/mod.rs:43`,

   registry retains no decode hook, so the kernel cannot actually dispatch

   is the explicitly-deferred Phase-1 work (kind-wrappers.md §6 + §8), called
   out in the merge commit body. Not a regression; tracked for Phase 1.**

2. **D1 violation — `ArticleDetailPayload.article: Option<_>`.**
   `crates/nmp-nip23/src/view/detail.rs:23`: detail rendering returns
   `Option`, not a placeholder/resolution-state payload. D1 says best-effort
   rendering with placeholders. **Genuine doctrine finding — orchestrator must
   decide: in-scope T79 fix-forward, or separate D1-conformance ticket.**

3. **Detail view state not scoped to its coord (real bug).**
   `crates/nmp-nip23/src/view/detail.rs:53`,
   `crates/nmp-nip23/tests/naddr_resolution.rs:103`: the `ArticleDetailView`
   accumulator is not filtered to the spec's `NaddrCoord`; an event for a
   different author with the same `d_tag` enters the state, and the test
   currently accepts Bob's record in a view opened for Alice's coord. The
   impl agent's earlier test "tightening" fixed the assertion vacuousness but
   NOT this underlying bug. **Genuine bug — needs a coord-filtering
   accumulator or coord stored in detail state. Orchestrator follow-up.**

4. **Replacement tie-break does not mirror store supersession.**
   `crates/nmp-nip23/src/domain.rs:117`,
   `crates/nmp-nip23/src/view/accumulator.rs:37`: NIP-33 replacement uses
   `existing.created_at >= record.created_at`; does not reuse the store's
   supersession comparator (equal-timestamp tie-break by event id). Suggested:
   reuse the store comparator + test stale/equal-ts events. Design-class
   refinement; orchestrator follow-up.

5. **Snapshot clones+sorts all records every emit (D8 risk).**
   `crates/nmp-nip23/src/view/accumulator.rs:69`: `snapshot_sorted()` is
   O(n log n) + full clone per emit. D8 wants bounded working set / zero
   per-event alloc after warmup. Suggested: maintain sorted state + emit
   deltas, add a reactivity-bench guard. Design-class; orchestrator follow-up.

6. **Key-encoding doc/impl mismatch.**
   `crates/nmp-nip23/src/domain.rs:58`: docs say "length-prefixed" keys; impl
   uses NUL (`\x00`) separators. A `d_tag` containing a NUL byte could
   collide. Suggested: switch to explicit length-prefix helpers before
   exposing `by_d_tag` scans; test separator bytes in `d_tag`. Correctness
   refinement (NUL in a NIP-23 `d` tag is pathological but not impossible);
   orchestrator follow-up.
