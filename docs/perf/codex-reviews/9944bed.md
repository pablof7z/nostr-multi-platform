## Report-only follow-ups (require orchestrator dispatch)

1. **`Nip07Signer::pubkey()` panics** (`crates/nmp-signers/src/signers/nip07.rs:65`) — `pubkey()` panics on a `new()`-constructed stub; `nip04()`/`nip44()` return `Some(self)` while only producing `Unsupported`.  Violates the trait shape "return `Some(_)` iff the signer can service that namespace."  Fix: redesign typed construction (`Nip07Signer::handshake().await -> Self` only callable on wasm; `Nip07Signer::from_cached_pubkey(pk)` for restored-from-payload), and have the trait methods return `None` when the namespace isn't actually supported.  Add tests for both code paths.

2. **NIP-46 plaintext-`encrypted_payload` shortcut** (`crates/nmp-signers/src/signers/nip46/mod.rs:228`) — current impl sends the raw `body_json` as the `encrypted_payload`.  This is intentional for unit-testability (the production transport encrypts), but the field name is misleading and there's no test asserting the transport is the one doing the encryption.  Fix: rename to `body_to_encrypt_and_publish` OR add a `Nip46Transport` integration test that uses a fake remote and round-trips through NIP-44 encrypt + sign + publish.

3. **NIP-46 response trusts remote `id`/`sig`** (`crates/nmp-signers/src/signers/nip46/mapper.rs:48`) — `parse_signed_event_response` accepts the returned `id` and `sig` verbatim and wraps the local `unsigned` template.  No verification that the returned `sig` validates against the recomputed event id.  Fix: compute the canonical event id from the returned envelope (not the local template — the remote might have legitimately massaged created_at), then verify the secp256k1 signature.

4. **`kind as u16` truncation** (`crates/nmp-signers/src/signers/local.rs:139`) — `UnsignedEvent.kind` is `u32` (kernel-side type); casting to `u16` truncates for kinds ≥ 65536.  Nostr kind space is `0..=65535` per the spec so this is currently safe, but the silent truncation is a latent footgun.  Also: malformed tags pass through `filter_map`, silently dropping anything that fails `Tag::parse`.  Fix: reject inputs that don't fit in u16 with a `SignerError::Backend` instead of truncating; reject malformed tag templates instead of dropping.  Add tests.

5. **Active-account-removal semantics** (`crates/nmp-signers/src/identity/manager.rs:171`) — removing the active account clears `active` but does NOT fire `ActiveChangeObserver`.  Downstream code that depends on rewire (kind:3, kind:10002 close-out) never gets the signal.  Fix: define removal semantics — either (a) require explicit `switch_active` to a different account before `remove` if removing the active, OR (b) fire `ActiveChangeObserver` with a synthetic "no active" event on removal-of-active.  Add a regression test.

6. **Soft-cap LOC overages** — `crates/nmp-signers/src/signers/nip46/mod.rs` (324), `tests/bunker_uri_fuzz.rs` (328), `tests/integration.rs` (342).  All under the 500 hard cap.  Report-only per the test-file precedent.

7. **PD-003 → PD-004 stale reference** (`docs/perf/m6/signers-impl.md:92`) — perf report still mentions `PD-003` for the ULID decision; renamed to `PD-004` in the ADR but missed here.  Codex left this as a doc-only follow-up at the 3-file fix boundary.
