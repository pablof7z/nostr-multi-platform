Fixes four issues flagged by post-merge codex review of bc84cfe:

1. `#p` inbox routing now uses `inbox_relays()` (read ∪ both) and
   `has_inbox_relays()` guard — a mailbox with only `both_relays` no longer
   falls back to indexer (NIP-65: `both` = read + write).

2. `request_probe` is now called in all fallback arms: Case A (unknown author),
   Case B (unknown address pubkey), and Case C (#p; already had it). Ensures
   mailbox cache is populated on next kind:10002 arrival.

3. Four-lane source discipline preserved: per-relay map in Case A now
   accumulates `BTreeSet<RoutingSource>` so relays reachable by both NIP-65
   (author A) and Indexer (author B) report both lanes in `role_tags`.
   `RelayEntry.source` replaced by `RelayEntry.sources`; `into_shape()` returns
   the full set; Stage 3 drains them all into `role_tags`.

4. `compile()` gains a prominent warning that its `CompileContext::default()`
   produces a plan-id that does NOT track indexer/config version changes.
   Production callers must use `compile_with_context`.
