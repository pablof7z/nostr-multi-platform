## Summary

Updates three documentation files to describe the new app-relay lane, the

`docs/design/subscription-compilation/outbox.md`, and

## Rubric

### Correctness

1. **Publish docs overstate shipped behavior** (HIGH): `docs/builder-guide/10-outbox-routing.md`
   states the indexer is used for discovery only (no longer a content fallback). This
   is true for subscription planning after T134, but the publish path
   (`Nip65OutboxResolver`) still falls back to the indexer when an author has no
   kind:10002. T134 only changed subscription planning, not the publish resolver.
   This creates a false contract for builders.

2. **Incorrect lane terminology** (MEDIUM): `SYNTHESIS-app-relays.md` references
   "Lane 7" as the shipped implementation. The actual implementation uses
   `UserConfiguredCategory::AppRelay` under `RoutingSource::UserConfigured`, not a
   numbered lane in the lane model. "Lane 7" is incorrect and misleads future readers.

3. **Case D cold-start under-documented** (MEDIUM): The indexer is still used in Case
   D (no-author / hashtag subscriptions) when both `active_account_read_relays` and
   `app_relays` are empty. This cold-start fallback is not documented in the updated
   docs, leaving the impression that the indexer is entirely removed from content
   routing.

4. **`apply_selection` note is forward-looking at this commit's point in time**: the
   docs note that `apply_selection` is wired into the lifecycle, but at f81f735 that
   wiring lands in 8fdc2ff (not yet merged). This is a doc/implementation
   ordering issue, not a permanent inaccuracy.

### Tests

Docs-only commit; no test changes. The accuracy issues are content bugs.

### Architecture Fit

Documenting the new model is correct and necessary. The errors undermine the value of
the docs as an authoritative contract.

### Follow-Ups

- Audit `Nip65OutboxResolver` and correct the publish-path claim or add a follow-up

- Replace "Lane 7" with correct terminology (`UserConfiguredCategory::AppRelay` /
  `RoutingSource::UserConfigured`).
- Document Case D cold-start indexer fallback explicitly.
