## Rubric

### Correctness

1. **RemoveRelay leaks the socket** (HIGH): `AddRelay` now calls
   `ensure_relay_worker()`, which spawns or returns an existing `RelayControl`, but
   `RemoveRelay` only mutates `relay_edit_rows` and never removes the `RelayControl`
   or sends `RelayCommand::Shutdown`. A user-removed relay stays connected until
   `Stop`/`Reset`/`Shutdown` (typically app restart). See `dispatch.rs:214` and
   `relay_mgmt.rs:195`.

2. **Integration path not tested** (MEDIUM): the new loopback test proves
   `ensure_relay_worker` dials (component level), and the command test proves
   `add_relay()` returns `Some`. But no test sends `ActorCommand::AddRelay` through
   the full actor dispatch path and asserts a socket is dialed. The key integration
   could regress while both existing tests still pass.

3. **URL canonicalization is only trim()** (MEDIUM): `relay_controls` is byte-string
   URL keyed, so `wss://r.example` and `wss://r.example/` are separate rows and
   sockets. This was a documented gap, but wiring user-configured sockets makes it a
   user-visible bug (user adds a relay, adds it again with a trailing slash, two
   sockets spawn). Promote to a follow-up task or fix the trim/normalization here.
   See `relays.rs:35` and `relay_mgmt.rs:70`.
