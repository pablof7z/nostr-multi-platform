## Summary

New example `crates/nmp-core/examples/outbox_perf.rs` (786 LOC) wires the
production `SubscriptionCompiler::with_relays()`, `apply_selection()`, and
`CompiledPlan::unroutable_authors`. Introduces `is_personal_relay()` (4-pattern
structural filter) and `parse_kind10002` (last-received-wins approximation).

## Rubric

### Correctness

1. **Last-received-wins bug** (HIGH): `parse_kind10002` is labelled "newest-wins
   approximation" in comments, but there is no `created_at` comparison. When multiple
   kind:10002 events arrive for the same pubkey the last to arrive wins regardless of
   timestamp. A stale event delivered out of order silently overwrites a newer one.

2. **Production ingest ignores personal-relay filter** (MEDIUM): `is_personal_relay()`
   is defined only in this example. The production ingest path
   (`kernel/ingest/relay_list.rs`) does not apply this filter, so personal-relay
   URLs (e.g. `wss://alice.nostr.me`) are included in NIP-65 write-relay sets and
   routed to for all followers — defeating the filter's purpose in real usage.

3. **File size** (MEDIUM): 786 LOC violates the 500-line hard ceiling from AGENTS.md.
   Split into `examples/outbox_perf/main.rs` + helper modules, or extract shared
   utility functions into a lib target.

### Tests

Tests pass but do not cover:
- Stale-event delivery order for `parse_kind10002`.
- Interaction between `is_personal_relay()` and the compiler's relay-routing output.
- Edge cases of `is_personal_relay()` pattern matching.

### Architecture Fit

Using production APIs (`SubscriptionCompiler::with_relays()`, `apply_selection()`,
`unroutable_authors`) in the example is good — it validates the integration. The
`is_personal_relay()` filter being example-only means production routing never
benefits from it; this split should be resolved deliberately.

### Follow-Ups

- Fix `parse_kind10002` to compare `created_at` and keep the event with the highest
  timestamp.
- Either move `is_personal_relay()` into `nmp-nip65` / `relay_list.rs` or document
  that it is intentionally example-only harness logic.
- Split `outbox_perf.rs` to comply with the 500-line hard ceiling.

## Findings

1. `parse_kind10002` is last-received-wins, not newest-wins — no `created_at`
   comparison; stale events delivered late silently overwrite fresh data.
2. `crates/nmp-core/examples/outbox_perf.rs` is 786 LOC, violating the 500-line
   hard ceiling in AGENTS.md.
3. `is_personal_relay()` exists only in the example; production
   `kernel/ingest/relay_list.rs` does not apply it, so personal relays are
   included in production NIP-65 routing.
4. Tests pass but do not cover stale-event ordering or `is_personal_relay()` edge
   cases.
