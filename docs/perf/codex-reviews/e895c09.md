# Codex post-merge review - PublishUnsignedEvent (e895c09)

## Verdict

PARTIAL.

Security claim verified: `PublishUnsignedEvent` does not let callers spoof authorship. The handler forwards the unsigned event to `sign_active`, `sign_with` signs with active `Keys`, and the returned `SignedEvent` writes `event.pubkey` from the signer, not caller `unsigned.pubkey`.

## Findings

- `crates/nmp-core/src/actor/commands/identity.rs:60` - High - REPORT: `UnsignedEvent.kind` is `u32`, but signing casts with `unsigned.kind as u16`. A malformed `kind:65559` would be signed and published as `kind:23` instead of rejected. Add range validation and surface the failure as toast state.
- `crates/nmp-core/src/actor/commands/identity.rs:64` - Medium - REPORT: malformed tag rows are silently dropped by `filter_map(|t| Tag::parse(t).ok())`. For a kind-agnostic publish pass-through, altered payloads should either fail as D6 state or be explicitly documented and tested as normalization.
- `crates/nmp-core/src/ffi/identity.rs:105` - Medium - REPORT: malformed JSON is defensive against panics, but it silently returns with no toast/state. `crates/nmp-testing/tests/publish_unsigned_event.rs:78` only tests serde round-trip, not `nmp_app_publish_unsigned_event`; `crates/nmp-core/src/lib.rs:30` test-support re-exports also omit the new FFI symbol, making actual Rust-side FFI coverage awkward.
- `crates/nmp-core/src/actor/commands/tests.rs:140` - Low - FIX: the security test used `ignored-by-signer` but did not assert active-signer pubkey provenance. Fixed in `b4914c3`.
- `crates/nmp-core/src/kernel/auth_tests.rs:456` - Low - FIX: existing clippy `get_first` lint in a test helper. Fixed in `b4914c3`.
- `crates/nmp-core/src/actor/commands/publish.rs:1` - Low - FIX: stale module docs did not mention generic unsigned publish. Fixed in `b4914c3`.
- `crates/nmp-core/src/actor/dispatch.rs:1` and `crates/nmp-core/src/actor/mod.rs:1` - Low - REPORT: 303 LOC and 412 LOC exceed the 300 LOC soft cap, but both remain below the 500 LOC hard cap.
