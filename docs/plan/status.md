# Historical Status Snapshot

> **Not authoritative.** Current planning/status truth lives in:
>
> - [`../plan.md`](../plan.md) — overarching milestone state and v1 exit criteria
> - [`../BACKLOG.md`](../BACKLOG.md) — active violations, pending decisions, and the ordered backlog
> - [`../../WIP.md`](../../WIP.md) — active branch/worktree tracker

This file used to be the per-milestone status ledger. It diverged from
`docs/plan.md` as the wasm, Android, desktop, routing, and crate-boundary work
landed, so it is no longer a source of current truth.

Historical references to `docs/plan/status.md` in older review and performance
notes should be read as pointers to the old snapshot that existed when those
notes were written. Do not add new status here; update `docs/plan.md` or
`docs/BACKLOG.md` instead.
