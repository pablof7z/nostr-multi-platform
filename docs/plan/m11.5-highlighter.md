# M11.5 — Highlighter rebuild + `nmp-nip29` crate

> Part of the [Build & Validation Plan](../plan.md). Arc 2 — kernel-boundary proof (second extension app).

**Demo product:** A 1:1 rebuild of `/Users/pablofernandez/Work/hl/app` (Highlighter — already has a Rust core + native UIs) running on NMP's substrate + codegen + view-module/action-module pattern. Builds the second non-social-domain extension app on top of the kernel, after the podcast app ([M11](m11-podcast.md)) proves the boundary.

**Why this milestone follows M11.** The podcast app proves the kernel boundary for audio/LLM/RSS domains. Highlighter proves it for a moderation-and-group-membership domain with a completely different data model (NIP-29 groups, kind 39000–39003 metadata events, group moderation flows). Two successful non-social rebuilds are the load-bearing evidence that the boundary is general, not podcast-specific.

**Scope.**

**Step 0 — survey and source mapping:**

- Read `/Users/pablofernandez/Work/hl/app` in full. Map every Swift/Kotlin view to its NMP extension-module equivalent. Identify what the existing Rust core already does vs what needs porting into the NMP substrate trait families.
- Produce `docs/design/highlighter-app-rebuild.md` before any porting begins.

**Step 1 — `nmp-nip29` protocol module crate:**

- Crate: `crates/nmp-nip29/`.
- Covers: groups (kind 39000), group metadata (39002), group admins (39001), group members (39003), NIP-29 moderation events (kind 9000–9009: add/remove user, edit metadata, delete event, create/delete group, join/leave requests).
- `GroupViewModule`, `GroupMembersViewModule`, `GroupFeedViewModule`, `GroupModQueueViewModule`.
- `JoinGroupAction`, `LeaveGroupAction`, `EditGroupMetadataAction`, `ModerateEventAction`, `CreateGroupAction`.

**Step 2 — `highlighter-core` app crate:**

- `apps/highlighter/highlighter-core/` — DomainModules, ViewModules, ActionModules mirroring the existing Rust core in `hl/app` but wired into NMP substrate trait families.
- Swift `Views/` copied verbatim → `// MARK: NMP-WIRE` markers → wired to Rust view modules.

**Exit gate (kernel boundary).**

- **`nmp-core` gains zero highlighter or group nouns.** Verified by grep + review.
- **`nmp-nip29` is a reusable protocol module crate** — any app can depend on it; it contains no Highlighter-specific business logic.
- **Reactivity, GC, diagnostics** behave identically to the Twitter slice and podcast app.

**Exit gate (product fidelity to `hl/app`).**

- All user flows from the Highlighter reference app reproduced with ≤ 1 px UI diff.
- Group moderation flows (add member, remove member, edit metadata) all round-trip via NIP-29 against a real NIP-29-capable relay.
- ADR-0007 diagnostics screen shows group subscriptions as logical interests, correctly scoped.

**Runnable artifact.** `ios/NmpHighlighter` — distinct binary, same Rust kernel, `nmp-nip29` protocol module. Report in `docs/perf/m11.5/highlighter-app.md`.
