- 2026-05-25 ‚Äî fix(nmp-cli): `nmp add component <id>` skips already-installed transitive dependencies instead of failing. The pre-fix loop rejected the whole install when ANY component in the resolved order (deps included) was already installed, so e.g. running `nmp add component swiftui/content-minimal` after `nmp add component swiftui/content-core` errored on the shared `content-core` dep. New regression test `add_component_skips_already_installed_dependencies` proves the fix by writing a sentinel into the dep's installed file between two invocations and asserting it survives. Existing `add_component_rejects_duplicate_installs` (same-target double-install) still passes ‚Äî that path is unchanged. `cargo test -p nmp-cli` (10 tests) + `cargo test -p nmp-testing --test doctrine_lint_smoke` (42 tests) green. ‚Äî branch `worktree-agent-ac351ff93200cd2a6`.
